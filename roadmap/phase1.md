# Phase 1 — Product Requirements

**Status:** Not Started
**Deliverables:** BRD, PRD, FRS, NFR, User Story Catalog

## Modules

### Module 1: Business Requirements
- [ ] Task 1.1: Capture business requirements from stakeholders
  - [ ] Subtask 1.1.1: Run requirements workshops
    - [ ] Activity: Schedule workshops with PMO, compliance, and engineering leads
    - [ ] Activity: Capture raw requirements notes
  - [ ] Subtask 1.1.2: Consolidate into a Business Requirements Document (BRD)
    - [ ] Activity: Draft BRD structure and fill in captured requirements
    - [ ] Activity: Get BRD sign-off from stakeholders

### Module 2: Product Requirements (PRD)
- [ ] Task 2.1: Translate business requirements into product requirements
  - [ ] Subtask 2.1.1: Draft PRD per core module (governance, workflow, audit)
    - [ ] Activity: Write PRD sections for Phase 5/6 modules
    - [ ] Activity: Review PRD with engineering for feasibility

### Module 3: Functional Requirements
- [ ] Task 3.1: Document functional requirements (FRS)
  - [ ] Subtask 3.1.1: Break down features into functional specs
    - [ ] Activity: Write FRS for RBAC, workflow engine, checklist engine
    - [ ] Activity: Cross-check FRS against BRD for coverage gaps

### Module 4: Non-Functional Requirements
- [ ] Task 4.1: Document NFRs (performance, security, scalability, availability)
  - [ ] Subtask 4.1.1: Define SLAs/SLOs
    - [ ] Activity: Set target p95 latency, uptime %, and audit-log durability requirements
  - [ ] Subtask 4.1.2: Define compliance NFRs
    - [ ] Activity: Map NFRs to ISO 27001 / SOC2 controls referenced in Phase 9

### Module 5: User Personas
- [ ] Task 5.1: Define primary personas
  - [ ] Subtask 5.1.1: Draft persona profiles
    - [ ] Activity: Write personas for PMO Lead, Governance Auditor, Delivery Manager, Developer

### Module 6: User Journeys
- [ ] Task 6.1: Map end-to-end journeys per persona
  - [ ] Subtask 6.1.1: Draft journey maps
    - [ ] Activity: Map "new project onboarding through governance sign-off" journey
    - [ ] Activity: Map "audit review" journey for the Auditor persona

### Module 7: User Stories
- [ ] Task 7.1: Write user stories for Phase 5/6 modules
  - [ ] Subtask 7.1.1: Draft stories in "As a ... I want ... so that ..." format
    - [ ] Activity: Write stories for RBAC and governance workflow
  - [ ] Subtask 7.1.2: Prioritize the story backlog
    - [ ] Activity: Run a prioritization session (MoSCoW or RICE)

### Module 8: Acceptance Criteria
- [ ] Task 8.1: Define acceptance criteria per user story
  - [ ] Subtask 8.1.1: Write Given/When/Then criteria
    - [ ] Activity: Attach acceptance criteria to each story in the backlog
